Project Viva Report
Driving School Management System

1. Project Overview

The Driving School Management System is a comprehensive, cross-platform mobile application designed to digitize and streamline the operations of a driving school. The system features role-based access for both Students (Users) and Administrators. It handles the end-to-end lifecycle of a driving school student, including course enrollment, test scheduling, retest management, secure payments, and performance tracking.

2. Technology Stack & Core Tools

  • App Framework: Flutter (Allows compiling to Android, iOS, and Web from a single codebase)
  • Programming Language: Dart (SDK Version 3.1.5+)
  • State Management: Provider (Used to cleanly separate UI presentation from business logic via controllers like UserController and AdminController)
  • IDE / Developer Tools: Visual Studio Code / Android Studio

3. Backend & Database Infrastructure

The project uses a Serverless Backend Architecture powered primarily by Firebase.

  • BaaS (Backend-as-a-Service): Firebase Core
  • Database: Cloud Firestore (A NoSQL, real-time document database used to store unstructured/semi-structured data like user profiles, booking details, test dates, and application statuses).
  • Media Storage: Cloudinary (cloudinary_public) is used as an external API to efficiently upload and serve images and media files (like user avatars or KYC documents), reducing the load on Firebase Storage.

4. APIs, Integrations, and Plugins

The project leverages several powerful APIs and third-party libraries to provide advanced features:

  • Authentication & Security API:
    • Firebase Authentication: Handles secure user registration and login.
    • Google Sign-In API: (google_sign_in) Allows users to log in quickly using their Google accounts (OAuth 2.0).
  • Payment Gateway API:
    • UPI India: (upi_india) Integrated to handle direct, seamless peer-to-peer UPI payments for course enrollments and retest fees without relying on a heavy payment gateway.
  • Machine Learning (AI) API:
    • Google ML Kit: (google_mlkit_text_recognition) Integrated for Optical Character Recognition (OCR). This allows the app to scan text from images (useful for scanning ID cards or driving licenses).
  • Notifications API:
    • Firebase Cloud Messaging (FCM): (firebase_messaging) Used to send push notifications to users regarding test date updates or application approvals.
    • Flutter Local Notifications: Handles the on-device display of these alerts.

5. UI/UX and Document Generation

  • Data Visualization: fl_chart is used in the Admin section to display interactive graphs and charts (e.g., tracking monthly revenue or student enrollments).
  • Document Generation: pdf and printing modules are utilized to dynamically generate, view, and print PDF documents (like payment receipts or completion certificates) inside the app.
  • Animations: lottie is used for high-quality, lightweight vector animations, and shimmer is used for modern loading effects.
  • Localization: easy_localization and intl are used to support multiple languages (e.g., English and Malayalam), making the app accessible to local users.

6. System Architecture Summary

The app follows a modern MVC/MVVM-inspired architecture:

  • Views: The UI screens (user_home.dart, my_applications.dart, test_management.dart) reactively rebuild when data changes.
  • Controllers: Logical classes (AdminController, UserController) that handle the heavy lifting—communicating with the database, manipulating lists, and updating the state.
  • Models: Data classes (like RetestModel or BookingModel) that map Firestore JSON data into Dart objects.

7. Expected Viva Questions & Answers

Q: Why did you choose Flutter over native Android (Java/Kotlin)?
A: Flutter allows us to write one codebase and deploy it to both Android and iOS, saving development time while maintaining native-like performance.
Q: Why use Cloudinary alongside Firebase?
A: Cloudinary provides excellent on-the-fly image optimization and transformations, which helps load images faster and saves bandwidth compared to raw Firebase storage.
Q: How does the state management work in your app?
A: We use the Provider package. When data changes in our backend (like an admin assigning a new test date), the controller updates the state and calls notifyListeners(), which instantly updates only the specific UI widgets relying on that data.
Q: How do you handle payments natively?
A: We integrated the upi_india package, which invokes installed UPI applications (like Google Pay or PhonePe) on the user's phone, completing the transaction securely bank-to-bank.
